# Lightweight Governance

Open-source governance does not need to be heavy to be useful. The goal is to make project decisions visible enough that contributors understand how work moves from idea to accepted change.

## What to Document

- Who can merge changes.
- How maintainers decide whether a proposal fits the project.
- Where roadmap decisions are recorded.
- How conduct and security reports are handled.
- When a decision should be revisited.

## Decision Logs

A decision log is a small file that records important project choices. It helps maintainers avoid repeating the same debate and gives contributors a source of context before they open a pull request.

Good decision log entries include:

- The date.
- The decision status.
- The context that made the decision necessary.
- Alternatives considered.
- How contributors are affected.
- A review date.

## Keep It Small

Start with a single `DECISION_LOG.md`. Add process only when the community needs it. A lightweight file that maintainers actually update is more valuable than a complex governance model nobody trusts.
