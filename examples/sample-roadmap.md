# Demo Community Roadmap

This roadmap uses fictional example data to demonstrate how Open Source Builder Kit can convert community health signals into practical planning artifacts.

## Roadmap Principles

- Protect maintainer time before adding new community programs.
- Make contribution paths explicit for both code and non-code contributors.
- Keep recommendations auditable by tying them to visible signals in manifests and demo inputs.

## 2026 Q3: Stabilize Contributor Operations

| Initiative | Projects | Outcome | Evidence Source |
| --- | --- | --- | --- |
| Maintainer guide templates | Civic Data Commons, Learning Lab Toolkit | Maintainers have documented triage, review, and release expectations. | Missing maintainer guide fields in manifests |
| Aging issue review | Civic Data Commons | Issues older than 90 days are closed, labeled, or assigned to a milestone. | Demo issue backlog signal |
| Seasonal review rotation | Learning Lab Toolkit | Classroom-focused pull requests receive faster review during planning months. | Demo median review time and maintainer count |
